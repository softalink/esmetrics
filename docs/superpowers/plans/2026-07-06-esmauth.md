# esmauth (vmauth port) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A standalone Rust auth proxy `esmauth` porting the open-source subset of VictoriaMetrics v1.146.0 `app/vmauth`: bearer/basic auth, path/host routing, load balancing across several backends, retries, backend health, global + per-user concurrency limits, and config hot-reload.

**Architecture:** A library crate `esm-auth` (config, auth, routing, balancing, proxy core — all unit-testable without sockets) plus a thin binary crate `esmauth` (flags, signals, wiring), built on the repo's existing synchronous `esm-http` server and blocking `reqwest` client (both already workspace deps). Two scoped additive extensions to `esm-http` give the proxy full-header capture and a streaming response pass-through.

**Tech Stack:** Rust 2021 workspace; `esm-http` (sync HTTP/1.1 server), `reqwest` (blocking, rustls — already a dep), a maintained serde-YAML crate (new dep, chosen in Task 1), `regex` (already a dep), `esm-common::metrics`.

## Global Constraints

- Upstream baseline **VictoriaMetrics v1.146.0** at `/home/test/refsrc/VictoriaMetrics`; sources `app/vmauth/{main,auth_config,target_url}.go`. `jwt.go`/`oidc.go` are OUT of scope. Algorithmic fidelity is the goal; port the upstream `*_test.go` cases as the primary safety net.
- **Security model (critical, get this exactly right):** vmauth does NOT compare secrets per request. At config load it builds a map keyed by the string `http_auth:<full-auth-header-value>` (see `getAuthTokens`/`getHTTPAuth*Token` in auth_config.go:1212-1251). At request time it derives candidate `http_auth:...` keys from the request and does a **map lookup** (`getUserInfoByAuthTokens`, main.go:218). There is no credential-comparison loop, so no timing side-channel on the compare itself — preserve this exact model; do NOT introduce a per-request string compare over the user list.
- **Security non-negotiables:** auth tokens NEVER logged unless `-logInvalidAuthTokens` is explicitly set (default off); bearer tokens/passwords NEVER used as `/metrics` label values (usernames MAY be — they are operator-chosen config identifiers); hop-by-hop headers stripped both directions; `src_paths` matching happens on the already-percent-decoded path exactly as upstream (esm-http already decodes `req.path()`).
- New workspace deps pinned in the root `Cargo.toml` `[workspace.dependencies]`; run `cargo tree -d` after adding to confirm no surprise duplication. Config atomic swap uses `std::sync::RwLock<Arc<AuthConfig>>` (no new dep) unless measured otherwise.
- Every crate builds warning-free on `x86_64-unknown-linux-gnu` and `x86_64-pc-windows-gnu`; MSVC is the release Windows artifact. `cargo fmt` + `RUSTFLAGS="-D warnings" cargo clippy --workspace --all-targets` clean. Files ≤ 800 lines (split by responsibility; tests to `#[path]` child modules or `tests/` when they push a file over).
- Exposed metric names use the `esmauth_` prefix (renamed from upstream `vmauth_`; mapping recorded in PORTING.md), consistent with the `esm_*` rename convention.
- Commit style `<type>: <description>`, no attribution trailers. After every push check CI (Linux + Windows MSVC) and fix failures before proceeding.

/// One backend: its URL string, an in-flight counter, and a "broken until"
/// instant. Port of backendURL (broken bool -> Instant-based fail window;

/// `fail_timeout` elapses, checked lazily on selection — simpler, and
