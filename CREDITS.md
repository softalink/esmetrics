# Credits

EsMetrics owes its existence to the design, documentation, and prior art of
several upstream projects. Each is acknowledged here.

## Reference projects

### VictoriaMetrics
- Project: <https://github.com/VictoriaMetrics/VictoriaMetrics>
- License: Apache License 2.0
- Reference pin: **v1.144.0** (released 2026-05-22)
- Relationship: EsMetrics is an independent Rust reimplementation of single-node
  VictoriaMetrics functionality. The on-disk format, HTTP API, CLI flag surface,
  and most algorithmic choices are designed to match VictoriaMetrics v1.144.0 so
  that EsMetrics can serve as a drop-in replacement. **No source code is shared
  between the two projects**; compatibility is achieved through clean
  reimplementation as permitted by the Apache 2.0 license.
- Trademark notice: "VictoriaMetrics" is a trademark of VictoriaMetrics Inc.;
  EsMetrics is not affiliated with or endorsed by VictoriaMetrics Inc.

### Prometheus
- Project: <https://github.com/prometheus/prometheus>
- License: Apache License 2.0
- Relationship: EsMetrics aims for PromQL semantic parity with Prometheus and
  uses Prometheus's `promqltest` corpus as a conformance test source.

### Facebook zstd
- Project: <https://github.com/facebook/zstd>
- License: BSD-3-Clause / GPLv2 (dual)
- Relationship: EsMetrics binds to the upstream zstd C library via the `zstd`
  Rust crate. Pinned to the version vendored by VictoriaMetrics v1.144.0 to
  guarantee byte-identical compressed part output.

## Vendored assets

### vmui (VictoriaMetrics UI)
- Source: VictoriaMetrics v1.144.0 release artifacts.
- License: Apache License 2.0.
- Relationship: EsMetrics embeds the upstream vmui static build unchanged via
  the `esm-vmui` crate. The crate's build script downloads and sha256-verifies
  the vmui artifact at build time, with a vendored fallback for offline builds.

cargo deny list
